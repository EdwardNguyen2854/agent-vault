# Vibe Coding vs Agentic Coding

I see **vibe coding** as the starting point.

You have an idea, open an AI coding tool, describe what you want, and keep adjusting until something works. It feels fast, creative, and a little chaotic — like sketching with code.

**Agentic coding** feels like the next level.

Instead of asking AI to only write code, you design a system where AI can plan, use tools, remember context, check files, run commands, and collaborate like a small team.

---

## Simple Difference

| Style          | Main idea                     | Best for                             | Risk                            |
| -------------- | ----------------------------- | ------------------------------------ | ------------------------------- |
| Vibe coding    | "Help me build this quickly"  | Prototypes, experiments, UI ideas    | Messy code, weak structure      |
| Agentic coding | "Help me operate this system" | Real projects, workflows, automation | Needs better design and control |

---

## Mental Model

```text
Vibe Coding
Idea → Prompt → Code → Test → Tweak

Agentic Coding
Goal → Plan → Tools → Memory → Execution → Review
```

---

## When to Use Each

**Vibe coding** — when you want to explore quickly, make a prototype, or care more about speed than structure.

**Agentic coding** — when the project is getting bigger, the AI needs context and memory, multiple steps/tools are involved, or you want repeatable workflows.

---

## Trade-off

```text
Speed        Vibe Coding     ██████████
Structure    Vibe Coding     ████

Speed        Agentic Coding  ███████
Structure    Agentic Coding  ██████████
```

---

## Final Thought

Vibe coding is like jamming with AI. Agentic coding is like building an AI-powered team.

Both are useful. But once a project becomes serious, the real skill is moving from "AI writes code for me" to "I design systems where AI can work responsibly."
