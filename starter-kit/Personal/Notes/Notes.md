# Windows Terminal Pane Management

If you work with multiple terminals — SSH sessions, dev servers, Docker logs, AI agents — you know the pain of scattered windows and constant Alt+Tabbing. Pane splitting solves this: one window, many terminals, no chaos.

Here's my go-to cheatsheet for Windows Terminal pane management.

---

## Shortcuts

The most important thing to learn is how to split, navigate, and close panes without touching the mouse.

| Action           | Shortcut                            |
| ---------------- | ----------------------------------- |
| Split vertical   | `Alt + Shift + +`                   |
| Split horizontal | `Alt + Shift + -`                   |
| Duplicate pane   | `Alt + Shift + D`                   |
| Focus left       | `Alt + Left`                        |
| Focus right      | `Alt + Right`                       |
| Focus up         | `Alt + Up`                          |
| Focus down       | `Alt + Down`                        |
| Close pane       | `Ctrl + Shift + W`                  |
| Zoom / unzoom    | `Alt + Shift + Enter`               |
| New tab          | `Ctrl + Shift + T`                  |
| Next / prev tab  | `Ctrl + Tab` / `Ctrl + Shift + Tab` |

The zoom shortcut (`Alt + Shift + Enter`) is the one I use most — it temporarily expands a pane to fullscreen so you can focus on one thing, then unzoom back to your layout.
