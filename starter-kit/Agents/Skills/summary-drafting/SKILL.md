# Summary Drafting

Create clear, concise summaries that capture note essence.

## When to Use

This skill activates when:

- User asks to summarize a note
- Drafting abstracts or descriptions
- Extracting key points from notes
- Improving note discoverability

## Workflow

### 1. Read the Full Note

- Understand context and purpose
- Identify the core idea
- Note the intended audience

### 2. Extract Key Points

- 3-5 main takeaways
- Important details or caveats
- Connected concepts or references

### 3. Write the Summary

- One sentence describing what the note covers
- 2-4 sentences of key points
- Related notes and links

## Summary Template

```markdown
## Summary

[One sentence describing what this note covers]

**Key points:**

- [Point 1]
- [Point 2]
- [Point 3]

**Related:** [[Note A]], [[Note B]]
```

## Principles

1. **Be factual** - no marketing or hype
2. **Be concise** - capture essence, not details
3. **Be useful** - help someone decide if they need this note
4. **Include context** - what makes this note worth reading?

## Anti-Patterns

- Vague descriptions: "This note is about X"
- Marketing speak: "Amazing insights into Y"
- Incomplete: "See the note for details"
- Spoilers: Giving away the conclusion before reading
